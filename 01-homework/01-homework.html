Перелік основних команд

  • git config --global user.name blabla - задати ім'я користувача
  • git config --global user.email blabal@bla - задати електронну адресу користувача
  • git init - створити Git-репозиторій
  • git status - показує стан репозиторію
  • git add - команда добавити усі файли в буфер
  • git commit - зміна коміта
  • git log - показує історії коммітів (в зворотньому хронологічному порядку)
  • git checkout - оновлення файлів в робочому дереві, щоб вони відповідали версії в індексі. Якщо специфікацію шляху не було вказано, git checkout також оновить HEAD, щоб встановити вказану гілку як поточну
  • git branch - управління гілками в репозиторії
  • git push - оновлення віддалених посилань, використовуючи локальні посилання
  • git pull - зміни з віддаленого сховища в поточну гілку
  • git merge - злиття окремих напрямів розробки, створених за допомогою команди git branch в єдину гілку
  • git clone - клонування сховища у щойно створений каталог, створює гілки віддаленого відстеження для кожної гілки в клонованому сховищі

.gitignore визначає навмисно невідстежувані файли, які потрібно ігнорувати. Файл gitignore визначає навмисно невідстежувані файли, які Git повинен ігнорувати. Файли, які вже відстежуються Git, не зачіпаються

Advanced

  • git revert - використовується для запису деяких нових комітів, щоб скасувати дію деяких попередніх комітів (часто лише несправних)
  • git reset - скидання поточного HEAD до вказаного стану
  • git rebase - повторно застосувати коміти поверх іншої базової підказки
  • git cherry-pick - застосувати зміни, внесені деякими існуючими комітами
  • git stash - дозволяє на якийсь час «здати в архів» (або відкласти) зміни, зроблені в робочій копії, щоб ви могли застосувати їх пізніше
